# OpenClaw Agenthon 2026 Technical Guidelines Summary

Source: http://ristek.link/Technical-Guidelines
Resolved document: Google Docs technical guidelines for **RISTEK x Build Club OpenClaw Agenthon 2026**.

## Competition format

- Online 12-hour AI Agent / Multi-Agent Systems build sprint.
- Participants may register individually or in teams of up to 4.
- Each participant/team may submit only one project.
- Teams may use any framework, AI model, API, programming language, or tooling.
- Hermes Agent, OpenClaw, Nano Claw, public/open-source templates, libraries, and third-party tools are allowed.

## Required agent characteristics

The project must include an AI Agent or Multi-Agent System capable of:

- reasoning;
- decision-making;
- tool usage / tool calls;
- workflow execution;
- autonomous operation through an autonomous loop;
- completing at least one task end-to-end without manual human intervention.

The project should not be merely a basic chatbot wrapper or an empty visual UI without functional AI backend logic. Judging emphasizes functioning agent logic and autonomy over pure visual polish.

## Submission requirements

Submission is through Devpost: https://openclawagenthon.devpost.com/

Required materials:

1. Project description.
2. Public GitHub repository with clear, reproducible README installation/run instructions.
3. Demo video uploaded to YouTube as Unlisted.
4. Pitch deck PDF.
5. Live deployment link, if available.
6. AI tools / models used.

Naming conventions from the guidelines:

- Team format: `OpenClaw2026_NamaTim`
- GitHub repository format: `OpenClaw2026_NamaTim_NamaProject`
- Demo video format: `OpenClaw2026_NamaTim_NamaProject`
- Pitch deck format: `OpenClaw2026_NamaTim_NamaProject.pdf`

## Demo video

- Maximum duration: 2 minutes.
- Must clearly show the product usage flow.
- Must show the AI Agent workflow clearly.
- Must be accessible to judges without permission requests.

## Pitch deck

- PDF only.
- Maximum 5 slides.
- Should cover:
  1. Problem Statement
  2. Solution Overview
  3. AI Agent Workflow / Architecture and Technical Explanation
  4. Key Features & Tech Stack
  5. Future Development / Impact

## Judging criteria

- Use Case Clarity & Impact: 10%
- Creativity & Originality: 30%
- Autonomy & Agent Behaviour: 30%
- Technical Execution: 20%
- Real-World Deployability: 10%

Optional additional track: **Best Payment Use Case**, for projects integrating payments or financial transactions.

## Ethics and compliance

- No plagiarism.
- No sabotage, exploitation, or harmful actions against participants or infrastructure.
- Maintain professional communication and conduct.
- Project ownership remains with participants, while organizers may display submissions/documentation/demos for event publication.

## RDTII Framework Agent interpretation

For this RDTII-focused Hermes Agent profile, emphasize:

- end-to-end autonomous RDTII policy review workflow;
- demonstrable tool calls for document ingestion, evidence extraction, mapping, scoring, and report generation;
- clear architecture showing Hermes Agent + RDTIIAnalyzer skill + reviewer-facing outputs;
- reproducible README and a concise demo that proves agent autonomy, not just chat.
